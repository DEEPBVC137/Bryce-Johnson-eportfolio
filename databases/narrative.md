# Databases Enhancement Narrative

## Artifact Description

The artifact selected for this enhancement is a database project originally developed using MySQL during earlier coursework. The original work focused on creating basic table structures and executing SQL queries to analyze data. This included defining tables such as customers, as well as running queries that used joins, grouping, and aggregation to explore return data across different regions.

The original artifact primarily consisted of screenshots showing table creation, schema structure, and query results. These queries demonstrated the ability to retrieve and organize data, such as counting returns by state and calculating percentages of returns by product. While this work showed a foundational understanding of relational databases, it was limited to isolated queries and did not represent a complete or fully implemented database system.

This artifact reflects an early stage in learning database design, focusing on individual SQL operations rather than a full system integration.

---

## Justification for Inclusion

This artifact was selected because it demonstrates my foundational understanding of database concepts and provides a clear opportunity for meaningful enhancement. The original work shows familiarity with SQL syntax, relational tables, and basic data analysis techniques, which are essential skills in database development.

However, the original implementation was limited in scope. It relied heavily on static queries and did not include a fully structured relational database or a system for managing data dynamically. Because of these limitations, this artifact was a strong candidate for improvement.

By enhancing this project, I was able to expand beyond simple query execution and develop a more complete database system. This allowed me to demonstrate growth in database design, data integration, and practical application of SQL in a more realistic environment.

---

## Enhancement Overview

The enhancements made to this artifact focused on transforming the original work into a more complete and functional database system.

One of the most significant improvements was the development of a structured relational schema. Instead of working with isolated tables, the enhanced version includes multiple related tables such as customers, orders, products, and returns. These tables are connected using primary and foreign keys, creating a more realistic representation of a database system.

Another major enhancement was the integration of real data using the Olist e-commerce dataset. A Python script was developed to import data from CSV files into the MySQL database. This allowed the system to move beyond static examples and work with a larger, more realistic dataset.

Additional SQL queries were created to perform more advanced analysis. These include multi-table joins, aggregation functions, and calculations that generate insights such as return rates and trends across products and regions. These queries demonstrate a deeper understanding of how data can be used to support decision making.

Screenshots were also included to show the database in operation, including populated tables and query results. These provide visual confirmation that the system is functioning as intended.

---

## Reflection on the Enhancement Process

Through enhancing this artifact, I developed a stronger understanding of how databases are designed and used in real world applications.

One of the main challenges was transitioning from working with individual queries to designing a complete relational system. This required a better understanding of how tables relate to each other and how to structure data efficiently using keys and relationships.

Working with a larger dataset also introduced new challenges. Importing data required careful handling to ensure that the data was properly formatted and inserted into the correct tables. This process helped me better understand how data flows into a database and how to manage it effectively.

Another important takeaway was the value of organization and planning in database design. Creating a clear schema before inserting data made it much easier to build queries and maintain consistency across the system.

Overall, this enhancement helped shift my perspective from simply writing SQL queries to thinking about databases as complete systems that store, manage, and analyze data.
