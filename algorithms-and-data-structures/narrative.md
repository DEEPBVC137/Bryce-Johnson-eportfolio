# Algorithms and Data Structures Enhancement Narrative

## Artifact Description

The artifact selected for this enhancement is the Pet Boarding System, originally developed in Java. This program models a simple boarding system for pets using object oriented programming principles. It was created earlier in the program to demonstrate foundational concepts such as classes, inheritance, and polymorphism.

The original version of the program consists of several core classes, including a base Pet class and derived classes such as Dog and Cat. These classes define attributes such as name, age, number of days staying, and owner information. The Dog class extends the Pet class by adding a grooming attribute, demonstrating inheritance and specialization.

The program is executed through the BoardingDriver class, where objects are manually created and displayed using method calls. Each object stores its own data and uses methods to output that data in a readable format.

This artifact represents a foundational understanding of data structures through class based design and demonstrates how object oriented programming can be used to organize and manage related data.

---

## Justification for Inclusion

This artifact was selected because it provides a clear example of early development in algorithms and data structures. It demonstrates my initial understanding of how to structure data using objects and how to apply core object oriented principles such as encapsulation, inheritance, and polymorphism.

The project also presents a strong opportunity for enhancement. While the original implementation effectively models individual pet objects, it lacks the ability to manage collections of data dynamically. All objects are hardcoded in the main method, which limits scalability and flexibility.

Because of these limitations, this artifact was a good candidate for demonstrating growth. It allowed me to expand beyond basic object oriented design and incorporate more advanced data structures and algorithmic thinking into the system.

---

## Enhancement Overview

The enhancements made to this artifact focused on improving how data is stored, managed, and processed within the system.

One of the most important changes was the introduction of a dynamic data structure, such as an ArrayList, to store multiple pet objects. This replaced the original approach of hardcoding individual objects in the main method. By using a collection, the program can now manage an expandable list of pets, making it more flexible and scalable.

A new management layer was introduced to organize operations on the data. This includes functionality for adding pets, storing them in the collection, and iterating through the list to display information. This change improves the overall structure of the program and separates responsibilities more clearly.

Additional improvements were made to support more advanced operations. These include the potential for searching for pets based on specific criteria, such as name or owner, and organizing data more efficiently. While the original program only displayed static data, the enhanced version moves toward a more interactive and functional system.

These enhancements transform the program from a simple demonstration of object oriented programming into a more complete system that better reflects real world data handling.

## Reflection on the Enhancement Process

Through enhancing this artifact, I gained a deeper understanding of how data structures and algorithms contribute to building scalable applications.

One of the main challenges was transitioning from a static design to a dynamic one. The original program worked correctly, but it did not handle data in a way that could grow or adapt. Introducing a collection required rethinking how data is stored and accessed throughout the program.

This process reinforced the importance of choosing the right data structure for a given problem. Using an ArrayList made it easier to manage multiple objects and iterate through them, but it also highlighted how different structures could be used depending on the needs of the application.

I also learned the importance of separating responsibilities within a program. By introducing a management class or structured approach to handling data, the code becomes easier to read, maintain, and expand. This reflects a shift from writing simple programs to designing more organized systems.

Throughout the enhancement process, I applied feedback from earlier coursework and focused on improving both functionality and structure. This helped me better understand how algorithmic thinking supports efficient and maintainable software design.
